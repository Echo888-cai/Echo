# Echo Editorial UI

Echo 的界面语言服务于一个目标：让复杂研究显得清晰、可信、可复盘。设计以编辑出版物的排版秩序为骨架，以 Apple 原生软件的克制交互为参照，不用装饰制造“专业感”。

## 设计原则

1. **证据先于容器**：优先用排版、分隔线和留白组织内容；只有输入、浮层和确有边界的工作区才使用表面。
2. **单一强调色**：系统蓝只表示可操作、选中、聚焦和进度，不承担装饰作用。
3. **功能性玻璃**：玻璃只用于登录、输入工作区和浮层等需要前后层级的场景；普通信息卡使用平面暖白。
4. **动效解释状态**：Tab 指示位置、注册字段展开、提交进度和浮层出现可以运动；循环动效仅允许极弱环境光，并必须尊重 `prefers-reduced-motion`。
5. **金融信息稳定**：数字使用等宽数字特性，涨跌不只依靠颜色，表格与估值区避免浮动和夸张缩放。

## 设计令牌

令牌定义在 `crates/echo-web/style.css` 的 `Echo Editorial System` 区域，组件不得自行发明新的品牌色。

| 用途 | 令牌 | 默认值 |
| --- | --- | --- |
| 主文字 | `--ink` | `#1d1d1f` |
| 次级文字 | `--ink-2` | `#3a3a3c` |
| 辅助文字 | `--muted` | `#6e6e73` |
| 系统蓝 | `--accent-strong` | `#0071e3` |
| 页面底色 | `--canvas` | `#f5f5f7` |
| 细分隔线 | `--hairline` | `rgba(0,0,0,.075)` |
| 错误 | `--danger` | `#c93400` |

圆角分为 `10 / 14 / 20 / 26px` 四级；阴影只使用 `--sh-1`、`--sh-2`、`--sh-3` 和浮层专用 `--sh-float`。

## 组件交互

### 主按钮

- 默认使用纯系统蓝，不使用多色渐变。
- Hover 只上移 `1px` 并略微加深；按下缩放到 `0.985`。
- 禁用态降低不透明度并使用 `not-allowed`，加载态必须同步 `aria-busy`。
- 按钮文案描述结果，例如“进入研究台”“保存偏好”，不使用含糊的“确定”。

### Tab 与分段控制

- 页面级 Tab 使用文字加底部指示线；容器级分段控制使用低对比灰底。
- 选中状态同时提供颜色、位置和 `aria-selected`，不能只靠颜色。
- 指示线在 `280ms` 内完成，不做弹跳或大幅缩放。

### 输入

- 默认高度 `46–48px`，边框使用中性黑色透明度，不使用彩色描边。
- Focus 使用系统蓝边框和低透明度四像素光环。
- 校验规则在提交前可见；错误信息使用 `role="alert"`，切换模式或重新输入时清理过期错误。
- 密码显示按钮必须有 `aria-label` 与 `aria-pressed`。

### 表面

- 普通卡片：暖白、细边框、`--sh-1`，Hover 最多上移 `1px`。
- 输入工作区与浮层：允许单层背景模糊，不嵌套玻璃卡片。
- 不使用成排彩色胶囊、装饰性高光边和重复渐变。

## 响应式与可访问性

- 桌面登录页为品牌叙事加窄表单双栏；`760px` 以下改为单栏，表单最大宽度 `400px`。
- 所有操作必须支持键盘；焦点环不可移除。
- 触控目标建议不小于 `44px`。
- 文字与背景维持可读对比度；状态不能只依赖颜色。
- `prefers-reduced-motion` 下关闭环境漂移、扫光与位移动画。

## 新增界面检查清单

- 是否能通过排版和留白解决，而不是新增卡片？
- 是否只使用了设计令牌？
- Default、Hover、Focus、Active、Disabled、Loading、Error 是否齐全？
- 键盘、触控、窄屏和 reduced-motion 是否可用？
- 数字、来源、时间和状态在快速扫读时是否清楚？
