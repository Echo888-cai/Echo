研究正文在抛错前已经生成（腾讯毛利率/现金流等数字齐全）；SSE 路径会先推 token，再因 `persistResearch` 抛错发 `error` 事件，前端表现为「无法输出正确研究结果」。

## 1. P0 · 落库外键吞掉已生成正文 🔴

### 证据

- 库内 users：`local`、`__e2e__`（2 行）。**没有** `u_a410f11a8138`。
- live 默认用户此前是 `u_a410f11a8138`（与 `accounts:reset-owner` 写入的 owner id=`local` 不一致）。
- LIVE-001 抛错前 params 里已经带完整回答（毛利率 55.7%、经营现金流 1013.51 亿等），随后：

```text
Failed query: insert into "research_sessions" (... user_id ...)
params: ..., u_a410f11a8138, 0700.HK, ...
```

- `research_sessions.user_id` → `users(id)` 外键；无效 user 直接失败。
- `runResearch` / `runReport` 在 `persistResearch` 上**没有**捕获：落库失败 = 整轮失败。
- SSE（`apps/api/src/rest-routes.ts`）先流式推 token，再 `await runAsk`；persist 抛错走 `event: error`。

### 裁决

| 项 | 判定 |
| --- | --- |
| 期望 | 研究结果应返回给用户；落库失败最多丢会话/画像 |
| 代码错 | ✅ `persistResearch` 失败不应吞正文；QA 默认 user 过期 |
| 期望错 | — |
| 测不到 | — |

### 已做修复（本轮）

1. `packages/application/src/research.ts`：`runResearch` / `runReport` 捕获落库异常，**仍返回正文**（`sessionId`/`portrait` 可为 null）。
2. `scripts/quality/agent-qa/live.ts`：默认 `ECHO_QA_USER_ID` 改为 `local`。
3. `docs/qa/methodology.md`：同步默认用户说明。
4. 流式路径：`streamSafeResearchText` + `readStreamedCompletion` 不再把 `FALSIFIERS_JSON` 推给前端；前端 `streamCallbacks` 再挡一层。落库失败 + 机器行泄漏是「答完卡住、光标停在 JSON 行」的组合根因。
5. 对比 FactGuard：`mergeFactsRegistry` 把对比对象行情/财报并进登记表，避免 full 模式对诚实引用误打 hard → 再打一轮纠错模型（看起来像卡死）。
6. 同业缺 key 时文案改为「同业数据源未配置」，不再谎称「本市场没有」。
