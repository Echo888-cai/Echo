# Luvio PRD

## 产品定位

Luvio 是面向**港股 + 美股**长期价值研究的 AI 研究室。同一套连续对话，既能研究港股（腾讯、比亚迪…）也能研究美股（AAPL、NVDA…）。底层用 `src/market.js` 统一识别市场并为各数据源拼对 symbol：港股行情走腾讯、美股行情走 Finnhub/Alpha Vantage/Yahoo，美股基本面用 FMP 免费档拿到真实 EPS/FCF。

目标用户不是高频交易者，而是希望把研究变成长期资产的个人投资者、研究员和小型投资团队。

## 核心价值

- 降低港美股公司研究的启动成本。
- 把自然语言问题转成结构化研究任务。
- 强制区分证据、假设、估值和风险。
- 用投资备忘录沉淀用户自己的研究历史。
- 用组合纪律页减少无意识集中和情绪化操作。

## 关键页面

`/` 研究室：
- 首屏直接提供研究助手。
- 支持当前公司切换。
- 展示当前档案完整度和下一步动作。
- 次级信息收进研究对话、研究档案、证据假设、股票池。

`/product` 产品价值页：
- 不是传统落地页。
- 用产品预览和价值叙事说明为什么 Luvio 存在。
- 展示竞品启发、Luvio 独特性和提示词架构。

`/company/[ticker]` 公司研究页：
- 一页摘要、商业模式、财务质量、护城河、管理层。
- 估值框架、风险审查、多空辩论、来源材料、投资备忘录。
- 研究进度、假设账本和结论门禁固定在页面前段。

`/portfolio` 组合纪律页：
- 估算成本暴露。
- 单一仓位集中度。
- 缺失成本、持仓、投资理由的项目。
- 行业暴露。
- 纪律提醒。

`/watchlist` 关注列表：
- 公司、理由、成本价、持仓、状态。

`/memo` 投资备忘录：
- 保存和编辑每家公司研究结论。

`/settings` 设置：
- 模型密钥策略。
- 模型网关说明。
- 数据来源和主题语言。

## MVP 演示路径

演示问题：

```text
帮我分析 1316.HK 耐世特是不是值得长期研究。我成本价 4.9，持有 3000 股。
```

演示步骤：

1. 首页研究室提问。
2. Luvio 识别公司并拆解问题。
3. 打开公司档案。
4. 导入公告或年报文本。
5. 生成研究报告。
6. 输入估值假设。
7. 保存投资备忘录。
8. 在组合纪律页检查集中度和缺失上下文。

## 安全边界

- 不提供买卖建议。
- 不输出确定性股价预测。
- 缺数据时明确说明。
- 高风险结论需要反方风险审查。
- 所有研究输出都应保留来源状态。
